# UX Design Analysis Reference

This reference provides a set of core UX principles and a checklist for analyzing Figma designs.

## Core UX Principles

1.  **Clarity**: Is the purpose of the page and each element clear?
2.  **Consistency**: Are UI elements (buttons, typography, colors) consistent across the design?
3.  **Efficiency**: Can users complete their tasks with minimal effort?
4.  **Accessibility**: Is the design accessible to all users (e.g., color contrast, font size)?
5.  **Hierarchy**: Is there a clear visual hierarchy that guides the user's attention?

## Design Analysis Checklist

- [ ] **Navigation**: Is the navigation intuitive and easy to find?
- [ ] **Call to Action (CTA)**: Are CTAs prominent and their purpose clear?
- [ ] **Forms**: Are forms easy to fill out with clear labels and error messages?
- [ ] **Typography**: Is the text readable with appropriate font sizes and line heights?
- [ ] **Color Palette**: Is the color usage balanced and meaningful?
- [ ] **Whitespace**: Is whitespace used effectively to prevent clutter?
- [ ] **Responsiveness**: Does the design consider different screen sizes?

## User-Specific Directives

Always ask the user for any specific design systems (e.g., Material Design, Apple HIG) or brand guidelines they want you to follow. Incorporate these into your analysis.
